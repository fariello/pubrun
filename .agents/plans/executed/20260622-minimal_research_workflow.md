# Implementation Plan: Minimal Research Workflow Example

This plan details the addition of a complete, self-contained example workflow that models a real-world research analysis using `pubrun`, along with an automated pytest smoke test in the test suite to ensure the example continues to run and produce all expected artifacts.

All changes will be implemented **locally** first. No commits will be pushed to the remote repository.

---

## 1. Proposed Changes

### A. Minimal Research Workflow Example Component

#### [NEW] [analysis.py](file://~/VC/pubrun/examples/minimal-research-workflow/analysis.py)
A self-contained Python script implementing a scientific analysis:
- Generates synthetic 2D data (with noise) to satisfy the "synthetic data only" requirement (no internet/credential/external dependencies).
- Runs within execution phases (`pubrun.phase`) to structure the computational steps.
- Uses `pubrun.annotate` to inject metadata/hyperparameters (slope, intercept, sample size).
- Writes structured metrics using `pubrun.report` and raw regression outputs/predictions using `pubrun.artifact`.

#### [NEW] [README.md](file://~/VC/pubrun/examples/minimal-research-workflow/README.md)
Detailed walkthrough explaining:
- What the workflow does and how to execute it.
- **Reviewer notes** detailing:
  - Which fields in `manifest.json` are machine-specific / non-deterministic (hostname, paths, usernames, timing, git details) and will differ upon replication.
  - The secret redaction policy applied during capture.

#### [NEW] [expected_methods_text.md](file://~/VC/pubrun/examples/minimal-research-workflow/expected_methods_text.md)
Shows the exact "Computational Methods" paragraph generated when calling `pubrun methods` on the run directory created by the workflow.

#### [NEW] [manifest_excerpt.json](file://~/VC/pubrun/examples/minimal-research-workflow/manifest_excerpt.json)
An annotated JSON excerpt showing the structure of the key metadata sections (such as `phases`, `annotations`, `reports`, and `artifacts`) generated by this analysis.

#### [NEW] [generated_output_notes.md](file://~/VC/pubrun/examples/minimal-research-workflow/generated_output_notes.md)
Documents each file created inside the run directory (`manifest.json`, `stdout.log`, `evaluation_metrics.json`, `raw_data.csv`, `predictions.csv`) and its role in replication.

### B. Example Verification Integration

#### [MODIFY] [verify_all.py](file://~/VC/pubrun/examples/verify_all.py)
Update the master verification script to run the new workflow subdirectory:
- Run `analysis.py` inside the subdirectory.
- Verify clean exit code (0) and ensure output directory/files are created.

#### [NEW] [test_examples.py](file://~/VC/pubrun/tests/test_examples.py)
A pytest unit test that automatically runs in CI:
- Dispatches `examples/minimal-research-workflow/analysis.py` under a clean temporary directory structure.
- Asserts that it exits successfully (exit code 0).
- Verifies that all expected artifacts (`manifest.json`, `stdout.log`, `evaluation_metrics.json`, `raw_data.csv`, `predictions.csv`) are written.
- Inspects the generated `manifest.json` and asserts that all phases (`data_generation`, `model_fitting`, `model_evaluation`), report entries, and artifact files are correctly registered.

---

## 2. Verification Plan

### Automated Tests
Run the entire pytest suite, including the new example smoke test:
```bash
python -m pytest tests/test_examples.py -v
```

### Script Execution Check
Run the master examples verification script directly to confirm all examples pass:
```bash
python examples/verify_all.py
```
